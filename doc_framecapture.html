framecapture Documentation

Although the base functionalities are written in C++ , to help you get started using this plugin, I have set up in depth example in Blueprints that is going to help you understand how this system is working, and how to implement this in your project.

Getting Started

First ensure that the plugin is installed and enabled by navigating to your plugin window from Edit -> Plugins and verifying if the framecapture plugin is enabled.

The ViewportClient

You might be familiar with editor command that helps you take high res screenshot of your game at any point of time. To implement similar but customized feature, I had to create a seperate Viewport Client that is inherited from GameViewportClient. My class is called framecaptureViewportClient. So before you start implementing my plugin in your project, you need to add my viewport client class as your game viewport client.

Go to Edit -> Project Settings, and navigate to Engine -> General Settings. Then update your Game Viewport Client Class with framecaptureViewportClient. You might have to restart the editor for your changes to take effect.

The Plugin Structure

The Plugin Content directory contains example blueprints and widgets which is very useful as a starting point to understand the functinalities of the plugin as well as to test out the core features.

To view the Plugin Content directory, you need to click on the eye icon at the bottom right of the Content Browser, and enable Show Engine Content.
